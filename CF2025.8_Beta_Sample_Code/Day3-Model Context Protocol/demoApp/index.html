Tools - listTools() and callTool()

Tools are callable functions exposed by MCP servers. The model discovers available tools via listTools() and invokes them via callTool().

mcpClient = MCPClient({ transport: { type: "HTTP", URL: serverUrl }, clientInfo: { name: "demo-client", version: "1.0.0" } }) // discover available tools tools = mcpClient.listTools() // invoke a tool result = mcpClient.callTool({ name: "echoMessage", arguments: { messageText: "hello" } })

Resources - listResources() and readResource()

Resources are data the server exposes for reading - documents, files, database records. The client discovers them via listResources() and reads content via readResource(uri).

// discover available resources resList = mcpClient.listResources() // resList.resources -> array of { uri, name, mimeType } // read a specific resource by URI content = mcpClient.readResource( "file:///resources/logs/app.log" ) writeOutput( content.contents[1].text )

Prompts - listPrompts() and getPrompt()

Prompts are reusable message templates the server provides. Users select a prompt and fill in arguments; getPrompt() returns structured messages ready to send to an LLM.

// discover available prompts prompts = mcpClient.listPrompts() // prompts.prompts -> [{ name, description, arguments }] // get a filled prompt template prompt = mcpClient.getPrompt({ name: "explain_code", arguments: { code: "x = x + 1", language: "CFML" } }) // prompt.messages -> ready to send to an LLM

Logging - Server-to-Client Log Messages

Servers emit structured log messages to clients in real time. The client registers a loggingConsumer callback that receives log entries with level and data. Levels: debug, info, notice, warning, error, critical, alert, emergency.

mcpClient = MCPClient({ transport: { type: "HTTP", URL: serverUrl }, clientInfo: { name: "demo-client", version: "1.0.0" }, // register a callback to receive server log messages loggingConsumer: function(message) { // message.level = "info", "warning", "error", etc. // message.data = log payload from the server writeLog( text = "[MCP] " & message.level & ": " & message.data, file = "mcp-demo" ) } })

Sampling - Server Requests LLM Completion from Client

Sampling reverses the usual flow: the server asks the client to run an LLM completion on its behalf. The client registers a samplingConsumer callback; the server calls sampling/createMessage and the client returns the model's response.

mcpClient = MCPClient({ transport: { type: "HTTP", URL: serverUrl }, clientInfo: { name: "demo-client", version: "1.0.0" }, capabilities: { sampling: true }, // invoked when server sends sampling/createMessage samplingConsumer: function(request) { // request.messages -> messages to send to LLM // request.maxTokens -> token budget // call your LLM here and return the response return { role: "assistant", content: { type: "text", text: llmResponse }, model: "gpt-4o", stopReason: "endTurn" } } }) // Tool call triggers sampling/createMessage back to client result = mcpClient.callTool({ name: "send_sampling_request_for_summarizer", arguments: { text: "...", maxTokens: 150 } })

Elicitation - Server Asks the User for Input

During a tool call, the server may need information only the human can provide. It sends an elicitation request with a JSON schema describing the form fields. The client's elicitationConsumer returns the user's input.

mcpClient = MCPClient({ transport: { type: "HTTP", URL: serverUrl }, clientInfo: { name: "demo-client", version: "1.0.0" }, capabilities: { elicitation: true }, // callback invoked when server needs user input elicitationConsumer: function(request) { // request contains: message, requestedSchema // return action + content matching the schema return { action: "accept", content: { username: "Alice", email: "alice@example.com" } } } })

Roots - Scoping Server File System Access

Roots define which parts of the filesystem the client is willing to share with the server. The client registers roots via addRoot(); the server can then list files within those boundaries.

mcpClient = MCPClient({ transport: { type: "HTTP", URL: serverUrl }, clientInfo: { name: "demo-client", version: "1.0.0" }, capabilities: { roots: true } }) // register a filesystem root the server may access mcpClient.addRoot("file:///var/data/docs", "Documents") // verify registered roots roots = mcpClient.listRoots() // server can now list files within the root files = mcpClient.callTool({ name: "list_directory", arguments: { path: "/var/data/docs" } })

Basic Auth - Password-protected MCP Server

The server enforces HTTP Basic Authentication. The client encodes credentials into the Authorization header on every request.

// Server: require basicAuth serverConfig = { serverInfo: { name: "my-server", version: "1.0" }, capabilities: { tools: true }, tools: [{ cfc: "my.tools.Echo" }], basicAuth: { username: "mcp-client-user", password: "s3cr3tP@$$" } } McpServer(serverConfig).handleRequest()
// Client: attach Authorization header credentials = toBase64(username & ":" & password) mcpClient = MCPClient({ transport: { type: "HTTP", URL: serverUrl, headers: { "Authorization": "Basic " & credentials } }, clientInfo: { name: "my-client", version: "1.0" } })

CORS - Cross-Origin Resource Sharing

The server advertises allowed origins. Browsers send an OPTIONS preflight before the real request; the server responds with Access-Control-* headers granting or denying access.

// Server: enable CORS with allowed origins serverConfig = { serverInfo: { name: "my-server", version: "1.0" }, capabilities: { tools: true }, tools: [{ cfc: "my.tools.Echo" }], corsEnabled: "true", corsAllowedOrigins: [ "http://localhost:8521", "https://myapp.example.com" ] } McpServer(serverConfig).handleRequest()

maxRequestBodySize - Enforce Request Size Limits

The server rejects any incoming HTTP request whose body exceeds the configured byte limit. Useful for preventing oversized payloads from reaching tool handlers.

// Server: set a 512-byte request body limit serverConfig = { serverInfo: { name: "my-server", version: "1.0" }, capabilities: { tools: true }, tools: [{ cfc: "my.tools.Echo" }], maxRequestBodySize: 512 } McpServer(serverConfig).handleRequest()
