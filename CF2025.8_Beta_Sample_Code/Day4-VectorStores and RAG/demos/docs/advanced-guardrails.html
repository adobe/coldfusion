Input guardrails

Inspect every user message before it touches the retrieval and the model.

What this panel does

A custom CFC — guardrails/SensitiveDataGuardrail.cfc — is registered on the inputGuardrails list. For each message it returns { isPassed, failureMessage }. When a guardrail blocks the request the service returns the failure message instead of calling the LLM.

The code

rag = agent({
    chatModel : chatModel,
    retrievalAugmentor : { queryRouter: { contentRetrievers: [{ vectorStore: vs }] } },
    inputGuardrails : [ createObject("component", "guardrails.SensitiveDataGuardrail") ]
});

SensitiveDataGuardrail.cfc

component {
    public struct function validate(required string userMessage) {
        // Block if the message looks like an SSN or a credit card
        if (reFind("\b\d{3}-\d{2}-\d{4}\b", arguments.userMessage))
            return { isPassed: false, failureMessage: "Sensitive data detected: ssn" };
        return { isPassed: true };
    }
}

Panel flow

  • Safe questions (“How do I reset my password?”) pass the guardrail and get a normal RAG answer.
  • Questions containing an SSN are blocked — the UI tags them with the red BLOCKED BY GUARDRAIL badge.
