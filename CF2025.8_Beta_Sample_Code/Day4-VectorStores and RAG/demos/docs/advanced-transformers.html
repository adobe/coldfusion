Document & segment transformers

Custom UDFs that enrich your data as it flows through ingestion.

What this panel does

The runner registers two CFML functions on the ingestion pipeline: a documentTransformer that fires once per file, and a textSegmentTransformer that fires once per chunk. Both can write additional metadata that later ends up in the retriever’s results.

The code

rag = agent({
    chatModel : chatModel,
    ingestion : {
        source              : expandPath("../Documents/product-docs.txt"),
        documentSplitter    : { chunkSize: 400, chunkOverlap: 60 },
        vectorStoreIngestor : { vectorStore: vs },

        // Per-document enrichment
        documentTransformer : function(doc) {
            doc.metadata.source = "product-docs";
            doc.metadata.ingested = now();
            return doc;
        },

        // Per-segment enrichment
        textSegmentTransformer : function(doc, seg) {
            seg.metadata.length = len(seg.text);
            seg.metadata.hasPrice = findNoCase("$", seg.text) ? true : false;
            return seg;
        }
    }
});
rag.ingest();

Where the metadata shows up

  • Every stored chunk carries the extra keys written by your transformers.
  • The content injector can expose those keys to the LLM (see the Injector panel).
  • Useful for source tagging, timestamps, quality flags, or routing hints.
