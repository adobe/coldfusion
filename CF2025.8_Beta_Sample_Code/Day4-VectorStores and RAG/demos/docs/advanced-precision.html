Tuning retrieval with agent()

Live control over chunking and retrieval scoring.

What this panel does

The UI sends chunkSize, splitterType, maxResults and minScore as URL params to the init runner. The runner builds an agent() with those values and ingests product-docs.txt, so you can see how each knob changes the answer to the same question.

The code

rag = agent({
    chatModel : chatModel,
    ingestion : {
        source           : expandPath("../Documents/product-docs.txt"),
        documentSplitter : {
            splitterType : splitterType,   // recursive | sentence | paragraph | ...
            chunkSize    : chunkSize,
            chunkOverlap : 80
        },
        vectorStoreIngestor : { vectorStore: vs }
    },
    retrievalAugmentor : {
        queryRouter : {
            contentRetrievers : [{
                vectorStore : vs,
                maxResults  : maxResults,
                minScore    : minScore
            }]
        }
    }
});
rag.ingest();

Knobs exposed in the UI

KnobEffect
splitterTypeHow the document is segmented. sentence is sharp for FAQs; recursive is a safe default for mixed prose.
chunkSizeCharacters per chunk. Smaller → tighter matches, more chunks; larger → more context per hit.
maxResultsHow many chunks the retriever returns for each question.
minScoreSimilarity threshold. Raise it to filter out marginal matches.
