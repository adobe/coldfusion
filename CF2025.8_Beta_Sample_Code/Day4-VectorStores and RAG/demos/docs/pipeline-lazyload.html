Document Service · Lazy Load

documentService().lazyLoad()

Stream documents one at a time instead of buffering them all in heap.

What this panel does

Both panels read the same 50-article Corpus/ folder. The left panel uses load(), which returns a full array of documents. The right panel uses lazyLoad(), which returns an iterator — only the current document is held in memory while you process the corpus.

load() — eager

docs = docService.load({
    path    : "./Corpus/",
    pattern : "*.txt"
});

// All 50 documents sitting in heap before the loop runs
for (d in docs) { processDocument(d); }

lazyLoad() — streaming

iterator = docService.lazyLoad({
    path    : "./Corpus/",
    pattern : "*.txt"
});

// Peak memory ≈ 1 document, regardless of corpus size
while (iterator.hasNext()) {
    doc = iterator.next();
    processDocument(doc);
}

When to use which

MethodUsed for
load()Small, bounded document sets that comfortably fit in memory.
lazyLoad()Large corpora or unbounded folders where memory is the constraint.
