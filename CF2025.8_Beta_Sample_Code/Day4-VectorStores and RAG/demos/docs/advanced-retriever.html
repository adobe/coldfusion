Content retriever configuration

Fine-grained control over what the retriever pulls back for each question.

What this panel does

The runner configures a single contentRetriever with a tight similarity threshold and a specific description. The panel lets you see how maxResults and minScore shape the answer — too low and the model gets noise, too high and there is nothing to ground on.

The code

rag = agent({
    chatModel : chatModel,
    retrievalAugmentor : {
        queryRouter : {
            contentRetrievers : [{
                vectorStore : vs,
                maxResults  : 4,
                minScore    : 0.5,
                description : "Product documentation"
            }]
        }
    }
});

Retriever fields

FieldUsed for
vectorStoreWhich backing store this retriever queries.
maxResultsTop-k chunks returned per query.
minScoreSimilarity floor — chunks below this score are discarded.
descriptionHuman-readable label used by the query router to pick this retriever.
