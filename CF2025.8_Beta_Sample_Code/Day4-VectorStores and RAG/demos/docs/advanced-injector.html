Content injector

Decide exactly how retrieved chunks and the user’s question are assembled into the final prompt.

What this panel does

The runner wires a contentInjector with a strict prompt template that forces the answer into a SUMMARY / DETAILS / SOURCES layout, and exposes the file_name metadata on every chunk so the model can cite its sources.

The code

rag = agent({
    chatModel : chatModel,
    retrievalAugmentor : {
        queryRouter : { contentRetrievers: [{ vectorStore: vs, maxResults: 5 }] },
        contentInjector : {
            promptTemplate :
                "You are a support assistant. Format every answer exactly as:" & chr(10) &
                "SUMMARY: <one line>" & chr(10) &
                "DETAILS: - <fact> - <fact>" & chr(10) &
                "SOURCES: <file_name values>" & chr(10) &
                "Context (tagged with file_name):" & chr(10) &
                "{{contents}}" & chr(10) &
                "User question: {{userMessage}}",
            metadataKeys : ["file_name"]
        }
    }
});

Injector fields

FieldUsed for
promptTemplateFinal prompt sent to the model. Use {{contents}} for retrieved chunks and {{userMessage}} for the question.
metadataKeysMetadata fields to surface alongside each chunk (e.g. file_name, source).
