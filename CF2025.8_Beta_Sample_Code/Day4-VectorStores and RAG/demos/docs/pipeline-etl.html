Document Service · Ingest + Query

documentService() — the document toolkit

Load, transform, split, enrich and ingest documents as standalone steps — then let any RAG service query the result.

What this panel does

The left panel runs an ingestion job using documentService(). Each stage — load → transform → split → transformSegments → ingest — is an explicit call that you can schedule, log and inspect. The right panel builds an agent() pointed at the same Milvus collection and answers questions, with no re-ingestion on the query side.

What documentService() can do

documentService() is more than an ingestor. It gives you separate, composable operations over documents and segments, so you can mix and match them for loading corpora, running clean-up jobs, re-chunking an existing dataset, enriching metadata, or shipping a full end-to-end ingestion pipeline.

Demo code — left panel (ingest)

docService = documentService();

docs = docService.load({ path: expandPath("../Documents/"), pattern: "*.txt" });

docs = docService.transform(docs, function(doc) {
    doc.metadata.pipeline = "nightly-ingest";
    return doc;
});

segments = docService.split(docs, { chunkSize: 400, chunkOverlap: 60 });

segments = docService.transformSegments(segments, function(doc, seg) {
    seg.metadata.quality = len(seg.text) > 150 ? "high" : "low";
    return seg;
});

docService.ingest(segments, vs, { batchSize: 50 });
docService.close();

Demo code — right panel (query)

// Same vs object, no ingestion here
rag = agent({
    chatModel : chatModel,
    retrievalAugmentor : {
        queryRouter : { contentRetrievers: [{ vectorStore: vs, maxResults: 4 }] }
    }
});

rag.chat("What are the pricing plans?");

All documentService() methods

MethodWhat it does
load()Eagerly read files or folders into Document objects. Good for small to medium corpora.
lazyLoad()Stream documents from disk so you never hold the whole corpus in memory — use this for large datasets.
transform()Run a CFML UDF on each Document to clean text, add tags, attach metadata, or drop unwanted docs.
split()Chunk documents using a chosen splitter (recursive, sentence, paragraph, …) with a configurable chunk size and overlap.
transformSegments()Run a UDF on each chunk after splitting — add quality scores, source labels, or filter segments before they hit the embedding model.
ingest()Embed the segments and write them to a vector store, in configurable batch sizes.
ingestAsync()Same as ingest() but non-blocking — returns a handle so your request thread can move on while the job runs.
close()Release the service and any underlying resources when the job is done.

Mix and match

You don't have to run every stage. Each method is independent, so you can:

  • Use load() + transform() only, to clean and re-save a corpus.
  • Use split() + ingest() on documents you already have in memory.
  • Use lazyLoad() + ingestAsync() to ingest a large corpus in the background.
  • Re-run split() with a different chunkSize to re-chunk an existing dataset.
