Advanced RAG · Query Routing

Routing to the right knowledge base

Multiple content retrievers, one routing model, per-question dispatch.

What this panel does

The runner ingests each source into its own Milvus collection — product docs, the HR handbook and the FAQ — then registers them as three separate retrievers. A routingModel inspects each incoming question and picks the retrievers whose description best fits it.

The code

rag = agent({
    chatModel : chatModel,
    retrievalAugmentor : {
        queryRouter : {
            routingModel : chatModel,
            contentRetrievers : [
                { vectorStore: vsProduct,  description: "Product features, pricing, integrations" },
                { vectorStore: vsHandbook, description: "Employee policies, leave, expenses" },
                { vectorStore: vsFAQ,      description: "Customer support FAQs" }
            ]
        }
    }
});

How routing decides

  • The description on each retriever is what the routing model reads.
  • Questions can route to one retriever, several, or none — there is no forced pick.
  • Use a dedicated, cheap model for routing to keep latency low.
